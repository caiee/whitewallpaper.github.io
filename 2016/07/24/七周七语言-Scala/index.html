Posted on 2016-07-24   |   In Scala   |  

Scala 与 Java 关系

  • Scala 运行在 虚拟机上， 使得 Scala 可以和现存的应用同时运行
  • Scala 可以直接使用 Java 类库
  • 都是静态类型语言
  • 语法较接近
  • Scala 既 支持 面向对象范型， 也支持 函数式编程范型

不同之处

  • 类型推断， Scala 会尽可能 推断出 变量的类型
  • 函数式编程概念 高阶函数 (high order function)
  • 不变量 val
  • 高级程序构造 actor 模型， 作为一等对象类 (first-class) 的 XML 处理
  • 拥有 REPY， 测试简单方法时可以直接在命令行使用

从这句话可以看出， Scala没有像 Java 有那么多的 ceremony，

  • Java
    • 它更像是， 我有一套标准， 你按照我的标准来
    • 你想打印一个 Hello World， 没问题，先把 class 定义好， 再写 public static void main(String[] args) {}。
  • Scala
    • 你要输出什么东西？没问题，告诉我输出什么，我去做那些ceremony。

这也是理所应当的， 一个后诞生的语言在设计时必定会借鉴前一个语言不足的地方，并加以补足。

范围与元祖

Scala支持一等类型范围(Range)

Scala提供元祖。
元祖是一个固定长度的对象集合， 在许多函数式编程语言中也会有这个模式。
元祖的对象可以有不同类型， 使用元祖经常来表示对象以及它们的属性。
