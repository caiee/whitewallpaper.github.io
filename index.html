Posted on 2016-07-24   |   In Scala   |  

Scala 与 Java 关系

  • Scala 运行在 虚拟机上， 使得 Scala 可以和现存的应用同时运行
  • Scala 可以直接使用 Java 类库
  • 都是静态类型语言
  • 语法较接近
  • Scala 既 支持 面向对象范型， 也支持 函数式编程范型

不同之处

  • 类型推断， Scala 会尽可能 推断出 变量的类型
  • 函数式编程概念 高阶函数 (high order function)
  • 不变量 val
  • 高级程序构造 actor 模型， 作为一等对象类 (first-class) 的 XML 处理
  • 拥有 REPY， 测试简单方法时可以直接在命令行使用

Posted on 2016-07-09   |   In Concurrent   |  

任务执行

围绕 “ 任务执行 “ 构造并发应用程序：任务通常是一些抽象的且离散的工作单元，
可分解为多个任务， 简化程序的组织结构，提供一种自然的事务边界来优化错误恢复过程，以及提供一种自然的并行工作结构来提升并发性

在线程中执行任务

在设计结构时，先清晰的找出任务边界。
各个任务之间相互独立，不依赖于其他任务的状态、结果或边界效应。
独立性有助于实现并发。

正常的负载情况下， 服务器应用程序应该是 良好的吞吐量 和 快速的响应性 共存。

并发简史

为什么需要并发

充分 发挥 多处理器系统的强大计算能力。

  • 资源利用率
    • 同时运行多个程序， 提高资源利用率
  • 公平性
    • 粗粒度时间分片， 让用户和程序共享计算机资源，而非一个程序从头到尾运行，然后再启动下一个程序
  • 便利性
    • 为计算多个任务编写多个程序， 它们之间必要时互相通信

高效：在串行性和异步性之间找到合理的平衡。
